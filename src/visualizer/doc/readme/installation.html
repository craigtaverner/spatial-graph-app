Library Files

The yFiles for HTML library consists of the content of the lib directory, namely the JavaScript library modules in UMD format in lib/umd/yfiles, the same functionality in ES6 module library format in lib/es6-modules/yfiles, and the stylesheet lib/yfiles.css that is required for both of the former options.

In order to develop applications with yFiles for HTML, you need to provide valid license information. Please see the related section in the Developer's Guide for the necessary setup. Evaluation versions of yFiles for HTML come with a pre-generated evaluation license file that is located in demos/resources/license.js. You can include this file in any HTML page that uses the library.

For detailed descriptions see also the Setting up your Development Environment section of the yFiles for HTML Developer's Guide.

The top-level directory deployment contains the yFiles deployment tool that provides name obfuscation and minification especially for the yFiles for HTML library, amongst others. For more details, see the description in the deployment directory or the respective section in the Developer's Guide.

yFiles Developer Tools

Auto-completion support in your IDE can be enabled through the ide-support/yfiles-api.js file.

TypeScript support in your IDE can be enabled through one of the various ide-support/yfiles-api .d.ts file variants.

During development of your application, you can get runtime type information for library types and parameter lists of methods in your browser's JavaScript console by including the following JavaScript file: ide-support/yfiles-typeinfo.js

For detailed descriptions see also the yFiles Developer Tools section of the yFiles for HTML Developer's Guide.

In the debug package of yFiles for HTML, the lib.debug directory contains an unobfuscated version of the library. These files are not redistributable. They may be used for debugging purposes, only.
