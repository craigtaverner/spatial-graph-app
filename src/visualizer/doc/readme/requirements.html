yFiles for HTML Technical Requirements

Technical Requirements

Development

We suggest to use a modern IDE with HTML5 support like JetBrains WebStorm, JetBrains IntelliJ IDEA, Microsoft Visual Studio Code, NetBeans, Microsoft Visual Studio 2015, or Eclipse.

The Developer's Guide has detailed descriptions and tips for developing with yFiles for HTML in these IDEs.

Deployed Applications

Running yFiles for HTML applications, including the provided demo applications, requires an HTML5-capable browser with adequate SVG support.

The demos included in this package have been authored using ECMAScript 6. This means that running them directly will not work on any versions of Internet Explorer (but they work in Microsoft Edge).

The Deployment Tool in this package can be used to convert the demos to ECMAScript Level 5 so that they run on Internet Explorer back to version 9.

The online versions of the demos has been deployed using that tool and they can be experienced with older browsers, too.

ECMAScript 5

The library files themselves are ECMAScript 5 compatible and do not require a separate transpile step. In order to make full use of the library in Internet Explorer, an implementation of Promise objects is required, thought. A shim for Promises can be found in the library directory.

Supported Browsers

At the time of its release, this version of yFiles for HTML has been tested with the following browsers:

  • Internet Explorer 9, 10, 11
  • Firefox 40 and newer
  • Chrome 44 and newer
  • Microsoft Edge
  • Safari 8, 9, 10

You can find an updated list of compatible browsers for the most current version of yFiles for HTML on our website.
