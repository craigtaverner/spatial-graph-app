Description

This demo shows how to use yFiles for HTML in an Angular app (with Angular 2 and newer) using SystemJS.

Please note that for new projects, Angular CLI (which uses webpack) is recommended over SystemJS.

The following Angular features are shown:

  • The yFiles graph control is wrapped as an Angular component.
  • A custom node style renders nodes using Angular components.
  • The graph item data is provided as an injectable Angular data service.
  • A separate view shows the currently selected node using the same Angular component that renders the graph nodes.
  • Employee properties can be edited using an angular form component. The properties will update automatically in all views.
  • The currently selected item is propagated to the application component using an Angular event binding.
  • The currently selected item is bound to the separate node view and to the property form using an Angular property binding.

Angular CLI

Please note our Knowledge Base article Using yFiles in Angular CLI Projects if you are looking to integrate yFiles for HTML in an Angular CLI project.
